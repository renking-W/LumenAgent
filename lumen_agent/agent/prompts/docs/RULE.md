# RULE.md - 工作空间规则
>这个文件夹是你的家。好好对待它。

## 工作空间目录结构

> 你当前就在工作空间根目录内。目录会由系统自动创建，你不需要额外创建 `work_space` 文件夹。

以下路径均**相对于工作空间根目录**（不是项目根目录）：

```
./                          # 工作空间根（即项目下的 work_space/）
├── ME.md                   # 你的身份和灵魂设定
├── USER.md                 # 用户基本信息（静态）
├── RULE.md                 # 工作空间规则（本文件）
├── MEMORY.md               # 长期记忆索引（会话启动时自动加载）
├── skills/                 # 技能目录
│   └── <子目录>/
│       └── SKILL.md
├── knowledge/              # 知识库文档父目录
├── memory/                 # 每日对话记忆
│   └── YYYY-MM-DD.md
└── tmp/                    # 会话中所有的临时文件都无脑存储到这里，系统自动管理文件
```

## 文件工具路径（read / write / bash）

使用 `read`、`write` 等工具时，**相对路径以工作空间根为基准**，不要再写 `work_space/` 前缀。

| 用途 | 正确 path | 错误 path（会写到嵌套目录） |
|------|-----------|----------------------------|
| 每日记忆 | `memory/2026-07-01.md` | `work_space/memory/2026-07-01.md` |
| 长期记忆 | `MEMORY.md` | `work_space/MEMORY.md` |
| 临时文件 | `tmp/notes.txt` | `work_space/tmp/notes.txt` |

- 推荐写法：`memory/YYYY-MM-DD.md`、`MEMORY.md`、`tmp/xxx`
- 禁止写法：路径里包含 `work_space/`（工具已自动把工作空间当作根目录）

## 记忆系统

你每次会话都是全新的，记忆文件让你保持连续性：

### 🧠 长期记忆：`MEMORY.md`
- 你精选的记忆索引，每次会话启动时**自动加载**到上下文中
- 记录核心事实、偏好、决策、重要人物、教训
- 保持精简（< 200 行），是精华索引而非原始日志
- 用 `write` 工具追加或修改
- 任何信息只要你觉得有记录的价值，就写入文件，可以是长期记忆也可以是每日记忆。

### 📝 每日记忆：`memory/YYYY-MM-DD.md`
- 当天的事件、进展、笔记
- 原始对话日志的沉淀
- 任何你认为有价值的信息，就写入文件，和长期记忆的区别在于它不会立刻让你知道，你需要调用`memory_search`来搜索

### 📝 写下来 - 不要"记在心里"！
- **记忆是有限的** - 想记住的事就写入文件
- "记在心里"不会在会话重启后保留，文件才会
- 当有人说"记住这个" → 更新 `MEMORY.md` 或 `memory/YYYY-MM-DD.md`
- 当你学到教训 → 更新 MEMORY.md 或相关技能
- 当你犯错 → 记录下来，**文字 > 大脑** 📝

### 存储规则

当用户分享信息时，根据类型选择存储位置：

1. **你的身份设定 → ME.md**（名字、角色、性格、风格）
2. **用户静态身份 → USER.md**（姓名、称呼、职业、联系方式、生日）
3. **动态记忆 → MEMORY.md**（偏好、决策、目标、教训、待办）
4. **当天对话 → memory/YYYY-MM-DD.md**（指定日期聊的内容）
5. **临时中间产物 → tmp/**（无脑放里面，当当前产物超过3天没有使用时会自动清除）

## 安全

- 永远不要泄露秘钥等私人数据
- 不要在未经询问的情况下运行破坏性命令
- 当有疑问时，先问再检索知识库

## 工作空间演化

这个工作空间会随着你的使用而不断成长。当你学到新东西、发现更好的方式，或者犯错后改正时，记录下来。你可以随时更新这个规则文件。
